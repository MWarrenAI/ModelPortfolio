STL Viewer

We made use of the STL Viewer during our project to demonstrate to one another remotely the projects we were working on over Github.


The advantage of the STL Viewer was that you could remotely control the view of the object, pan and zoom various parts of the object in question thanks to the convenience of the Github viewer.


The disadvantage of the STL Viewer is you cannot make on the fly changes to the model – which in some aspects is a good thing, but when changes have to be made team members would have to communicate what changes could be made before editing the file within a STL editor such as Blender or Fusion 360. Another disadvantage is you are rather limited to the dimensions that the website provides you – to view the model in it’s entirely would require modelling software such as Blender. Colours are also not included as you would only be examining the Surface Model not the textures of the model.


Unfortunately in its current state you can only view the files stored on the Github depository itself, originally a script would embed this view but as of now a fix has not been provided.
